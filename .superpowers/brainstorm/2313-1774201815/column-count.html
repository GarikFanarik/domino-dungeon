How wide should the map be?

Fixed column count — nodes snap to these slots. Fewer = more forced branching. More = wider paths, more choice.

A — 4 Columns

Tight. Paths converge quickly, fewer choices per row. More intense — every decision matters more.

B — 6 Columns (recommended)

Sweet spot. Enough width for interesting paths without feeling sparse. Matches the current 3–5 node rows comfortably.

C — 8 Columns

Wide, airy. Lots of path variety and choice. Nodes are smaller to fit. Better for players who like planning routes far ahead.

Recommendation: B (6 columns) — best balance of route variety and readability at 1920×1080.
